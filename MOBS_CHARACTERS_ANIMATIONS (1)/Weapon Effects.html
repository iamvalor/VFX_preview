09 Hand-off · Implementation

For Claude Code · Compose Canvas

File plan

  • New — ui/screens/WeaponEffects.kt — holds all 8 DrawScope.drawXxx(state) functions plus the small easing helpers (easeOutCubic, easeOutQuint, easeOutBack).
  • Modify — GameScreen.kt · drawAttackVisuals(state) — replace the current 3-way when with the 7-way dispatch shown below. The existing 3 functions stay; they just call into the new ones.
  • Untouched — Weapon.kt, Enemy.kt, Player.kt, GameLoop.kt, CollisionDetector.kt, PlayerStats.kt. No gameplay edits.

Hard constraints

  • Every DrawScope.drawXxx(state) below is read-only against state — it must never write a value back.
  • Use state.attackProgress as t ∈ [0,1]. Never alter the gameplay's progress curve.
  • Sparks and trail particles are visual only — they don't deal damage.
  • If a weapon's shape changes shape arguments (e.g. arc widens), the visual will scale automatically because every dimension is derived from state.attackReach / state.attackArc / shape.*.
