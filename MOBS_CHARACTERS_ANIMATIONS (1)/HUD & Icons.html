HUD // ICONOGRAPHY

A consistent icon set and HUD layout for the neon-cyberpunk run, designed to drop straight into Compose Canvas. Tokens lifted from GameTheme.kt.

01 Principles

Stylized 2D vector · Hades-lite + cyberpunk
01

Glyphic silhouette

One bold shape per icon, readable at 22 px. No internal anatomy details.

02

One stroke weight

Every icon uses a 2.5px stroke in the 64-viewBox. Round caps.

03

Owned accent

Each icon carries one neon accent from the existing palette.

04

Subtle glow

SVG drop-shadow filter, same hue as the stroke, 4px blur, 60% opacity.

07 Implementation Spec

For Claude Code · Kotlin + Compose Canvas

File plan

  • New — ui/components/Icons.kt — path data + drawIcon helper.
  • Modify — GameScreen.kt — replace emoji stubs with icon calls.
  • Untouched — Enemy.kt, Player.kt, Weapon.kt, GameLoop.kt.

Hard constraints

  • All Kotlin gameplay files unchanged.
  • HUD bar widths and stat ranges unchanged.
  • Weapon slot count and acquisition logic unchanged.
Hand-off

Each icon's SVG path is copyable directly from the page; implement into Compose Path calls with a 64→size scale factor.
