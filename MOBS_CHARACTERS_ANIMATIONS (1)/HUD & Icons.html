VISUAL SPEC · UI ICONS + HUD

HUD // ICONOGRAPHY

A consistent icon set and HUD layout for the neon-cyberpunk run, designed to drop straight into Compose Canvas. Tokens lifted from GameTheme.kt. Hand-off targets Claude Code or another implementing agent — every icon is a self-contained SVG below.

01 Principles

Stylized 2D vector · Hades-lite + cyberpunk
01

Glyphic silhouette

One bold shape per icon, readable at 22 px in the HUD bar. No internal anatomy details — read as a symbol, not a drawing.

02

One stroke weight

Every icon uses a 2.5px stroke in the 64-viewBox, scaled with the icon. Round caps. Solid fills only when needed for mass.

03

Owned accent

Each icon carries one neon accent from the existing palette. Stat icons borrow their semantic color; weapons use cyan by default, switch to their elemental hue on level-up.

04

Subtle glow

SVG drop-shadow filter, same hue as the stroke, 4px blur, 60% opacity. Replicated in Compose with Paint.setShadowLayer.

Spatial budget

  • Top bar height: 96 dp
  • Bottom controls: 140 dp safe-zone
  • Center play area: always >= 720 dp tall on smallest landscape device
  • Stat bar width: 320 dp (fixed) so it doesn't compete with gameplay
  • Damage numbers: world-space, scale 1.0 normal / 1.6 crit · TTL 0.8s

Z-order

  • 0 — Biome backdrop image (untouched)
  • 1 — Pickup radius rings, chain lightning radius
  • 2 — Particles, XP orbs, projectiles
  • 3 — Attack zones, enemies, player, attack visuals
  • 4 — Damage numbers, off-screen indicators
  • 5 — HUD overlay (all icons live here)
  • 6 — Pause modal / upgrade picker

07 Implementation Spec

For Claude Code · Kotlin + Compose Canvas

File plan

  • New — ui/components/Icons.kt — single object holding all path data + a drawIcon(icon, size, color) helper using Compose Path.
  • New — res/drawable/ — same icons as XML vectors (so they work in non-Canvas contexts: upgrade cards, settings).
  • Modify — GameScreen.kt — swap the existing emoji/text icon stubs in HealthBar, StaminaBar, XpBar, WeaponSlotsRow for Icons.draw… calls. No changes to bar widths, ranges, or values.
  • Modify — UpgradeScreen.kt — read the weapon's WeaponType and render its icon on the card.
  • Untouched — Enemy.kt, Player.kt, Weapon.kt, GameLoop.kt, CollisionDetector.kt, PlayerStats.kt. Zero gameplay edits.

Path-to-Compose translation

  • SVG viewBox is always 0 0 64 64. Translate by scaling: val s = size / 64f, then multiply every coordinate.
  • Strokes use Stroke(width = 2.5f * s, cap = StrokeCap.Round, join = StrokeJoin.Round).
  • For the neon glow: drawIntoCanvas { canvas -> canvas.nativeCanvas.drawPath(p, Paint().apply { setShadowLayer(8f, 0f, 0f, glowColor.toArgb()); strokeWidth = …; style = Paint.Style.STROKE }) }.
  • Fill alpha is always 0.10–0.15 of the stroke color — keeps icons readable on bright biomes.

Hard constraints (do not regress)

  • All Kotlin gameplay files — Enemy.kt, Player.kt, Weapon.kt, GameLoop.kt, CollisionDetector.kt, PlayerStats.kt — must stay byte-identical. No radius, speed, damage, cooldown or hitbox value changes.
  • The HUD bar widths and stat ranges in HealthBar/StaminaBar/XpBar are unchanged — we only swap the leading icon.
  • Weapon slot count and acquisition logic are unchanged — only the slot contents are re-skinned per WeaponType.
  • Upgrade rarity colors keep using RarityColors; we just re-tint the slot border instead of drawing a colored rectangle.
